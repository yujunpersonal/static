Политика конфиденциальности

Последнее обновление: 27 апреля 2026 г.

Настоящая Политика конфиденциальности объясняет, как мы собираем, используем и защищаем информацию, когда вы пользуетесь нашими приложениями, распространяемыми через Apple App Store.

1. Какие данные мы собираем и как мы их используем

Разные наши приложения собирают разные категории данных в зависимости от используемых вами функций. Категории, перечисленные ниже, описывают максимальный охват по всем нашим приложениям; отдельное приложение собирает только то, что необходимо для его функций.

  • Аудиозаписи. Записываются микрофоном вашего устройства только тогда, когда вы активно начинаете запись или голосовое взаимодействие в приложении. Используются для транскрипции сказанного вами и, при необходимости, для отправки вашего голосового запроса в сервис ИИ, чтобы приложение могло ответить.
  • Транскрипции и текстовые подсказки. Текстовая версия вашего голосового ввода, а также любой текст, который вы вводите в поле подсказки или чата. Используются для генерации ответа приложения и, если приложение поддерживает историю переписки, для отображения предыдущих взаимодействий на вашем устройстве.
  • Изображения и фотографии. Только конкретные изображения, которые вы выбираете в приложении (например, при выборе аватара). Используются исключительно для той внутриприложенческой цели, для которой вы их выбрали.
  • Информация о покупках и подписках. Предоставляется Apple StoreKit при совершении вами покупки. Мы получаем идентификатор транзакции и статус подписки; платёжные данные мы не получаем.
  • Локально хранимые настройки. Настройки приложения, выбор языка и (если применимо) ключ API, который вы предоставляете для стороннего сервиса ИИ. Они хранятся на вашем устройстве — настройки в системном хранилище параметров по умолчанию, а ключи API в iOS Keychain. Они не передаются нам.
  • Диагностика. Журналы сбоев и агрегированные обезличенные данные о производительности — только если вы включили это в Настройках iOS → Конфиденциальность и безопасность → Аналитика и улучшения.

Мы не собираем ваше имя, адрес электронной почты, список контактов, точное местоположение или идентификаторы устройства, используемые для рекламы. Мы не продаём ваши данные и не используем их для обучения каких-либо моделей машинного обучения.

2. Функции ИИ и сторонние поставщики ИИ

Некоторые наши приложения включают функции, основанные на сторонних сервисах искусственного интеллекта. Если приложение предлагает такую функцию, его внутреннее описание и запрос согласия сообщат вам, какой поставщик используется, до того как любые ваши данные будут отправлены.

В зависимости от приложения и выбранного вами поставщика следующие данные могут передаваться непосредственно с вашего устройства одному или нескольким из этих поставщиков:

  • Аудиозаписи, которые вы создаёте в приложении
  • Транскрипции вашего аудио
  • Текстовые подсказки, которые вы вводите
  • История переписки, необходимая для предоставления ИИ контекста для ответа

Сторонние сервисы ИИ, которым мы можем передавать эти данные:

  • OpenAI, L.L.C. — API распознавания речи, языковой модели и синтеза речи. См. политику конфиденциальности OpenAI по адресу https://openai.com/policies/privacy-policy и политику использования данных API по адресу https://openai.com/policies/api-data-usage-policies.
  • Anthropic, PBC — API языковых моделей (Claude). См. политику конфиденциальности Anthropic по адресу https://www.anthropic.com/legal/privacy.
  • Google LLC — API языковой модели Gemini. См. политику конфиденциальности Google по адресу https://policies.google.com/privacy и условия Google AI по адресу https://ai.google.dev/gemini-api/terms.

Если приложение поддерживает модель «принесите свой ключ» — то есть вы предоставляете собственный ключ API для поставщика ИИ — ваши данные отправляются непосредственно с вашего устройства этому поставщику в рамках вашей собственной учётной записи у него, и ваше использование регулируется вашим соглашением с этим поставщиком.

Мы изучили опубликованные политики каждого из этих поставщиков и, используя их API в рамках их коммерческих условий, требуем, чтобы они обеспечивали защиту ваших данных, эквивалентную описанной в настоящей политике. Согласно указанным выше политикам, эти поставщики по умолчанию не используют входные или выходные данные API для обучения своих моделей.

3. Ваш выбор и порядок удаления данных

  • Вы можете в любой момент остановить использование приложением функций ИИ, удалив ключ API в настройках приложения (если применимо) или отклонив запрос на согласие при его первом появлении.
  • Вы можете удалить всю историю переписки и записи, хранящиеся на вашем устройстве, в настройках приложения.
  • Вы можете в любой момент удалить приложение, что приведёт к удалению всех локально хранимых данных, включая любые сохранённые ключи API в Keychain.
  • Чтобы запросить удаление любых данных, которые мы храним о вас, свяжитесь с нами по адресу, указанному ниже.

4. Покупки в приложении и подписки

Платежи обрабатываются Apple, а не нами напрямую. Мы можем получать статус подписки и метаданные транзакции, необходимые для предоставления платных функций и поддержки клиентов.

5. Дети

Наши приложения предназначены для широкой аудитории и не направлены на детей младше 13 лет (или эквивалентного возраста в вашей юрисдикции). Мы сознательно не собираем личную информацию от детей младше 13 лет. Если вы считаете, что ребёнок предоставил нам личную информацию, свяжитесь с нами, и мы её удалим.

6. Хранение данных

Мы храним локальные данные приложения столько, сколько приложение установлено на вашем устройстве. Записи о покупках, связанные с вашим Apple ID, сохраняются в соответствии с требованиями Apple и применимого налогового законодательства. Мы не ведём для вас серверных учётных записей или профилей.

7. Безопасность

Предоставляемые вами ключи API хранятся в iOS Keychain, который шифруется операционной системой. Сетевые запросы к сторонним поставщикам ИИ выполняются по HTTPS.

8. Изменения в настоящей Политике

Мы можем периодически обновлять эту политику. Дата «Последнее обновление» выше отражает самое последнее изменение. Существенные изменения будут отражены в процедуре получения согласия в приложении при следующем запуске.
